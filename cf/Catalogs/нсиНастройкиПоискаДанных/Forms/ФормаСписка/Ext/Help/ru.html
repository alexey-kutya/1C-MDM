Справочник "Настройки поиска данных"

Система позволяет создавать и хранить сложные поисковые запросы. Это исходная информация для осуществления поиска с помощью функционала системы. Формат поискового запроса зависит как от устройства справочника НСИ, так и от типа информации для поиска. Поисковый запрос задаётся в виде набора полей справочника НСИ и информации, которую необходимо найти в данном поле.

Для задания нового поиска данных пользователь указывает:

  • наименование справочника, в котором будет осуществляться поиск;
  • и задает имя поискового запроса.

 

В зависимости от выбранного справочника система на панели «доступные поля» предложит список полей, по которым возможно осуществить поиск. Пользователь выбирает интересующие его атрибуты справочника двойным кликом мыши. Выбранные атрибуты справочника будут показаны в области «отбор». По каждому из атрибутов справочника можно указать вид сравнения (равно, начинается с, содержит, заполнено и др.) и значение, с которым будут сравниваться записи справочника.

В случае необходимости по некоторым атрибутам можно задать полнотекстовый поиск, отыскивая заданную строку с учетом поддержки транслитерации (написание русских слов символами латиницы в соответствии с ГОСТом); поддержка замещения (написание части символов в русских словах одноклавишными латинскими символами); нечеткий поиск (буквы в найденных словах могут отличаться). Для этого необходимо указать у нужного атрибута справочника метод сравнения данных и процент совпадения.

Система поддерживает следующие методы сравнения строк:

  • Левенштейна (по символам) - сравнение на неточное совпадение, где сравниваются строки с точностью до символа;
  • Левенштейна (по словам) - сравнение на неточное совпадение, где сравниваются строки с точностью до слова;
  • Дамерау-Левенштейна (по символам) - сравнение на неточное совпадение с учетом перестановок, где сравниваются строки с точностью до символа;
  • Дамерау-Левенштейна (по словам) - сравнение на неточное совпадение с учетом перестановок, где сравниваются строки с точностью до слова;
  • Метод «N-грамм» - сравнение на неточное совпадение, где сравнивается количество вхождение подстрок оригинала со сравниваемой строкой (длина подстроки указывается в настройках системы);
  • Метод «Биграмм» - сравнение на неточное совпадение, где сравнивается количество вхождение подстрок оригинала со сравниваемой строкой (длина подстроки 2 символа);
  • Метод «Триграмм» - сравнение на неточное совпадение, где сравнивается количество вхождение подстрок оригинала со сравниваемой строкой (длина подстроки 3 символа).

 

В системе отсутствуют ограничения на количество поисковых запросов созданных пользователем. Создав и сохранив поисковый запрос, пользователь может выбрать в меню работы с записями справочника действие «показать группу расширенного поиска», вызвать функционал расширенного поиска данных по атрибутам и характеристикам. На панели отбора необходимо выбрать нужный поисковый запрос и запустить процедуру поиска.
