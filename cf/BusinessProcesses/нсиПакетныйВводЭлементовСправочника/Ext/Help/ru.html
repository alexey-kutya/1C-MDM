Ввод нового элемента справочника

Добавление и изменение справочников в системе осуществляется по заявкам пользователей. В случае если пользователь не нашел в справочнике необходимый ему объект НСИ он формирует запрос в системе на добавление новой записи в справочнике. Формирование заявки на добавление новой записи возможно в системе несколькими способами. Вызвать форму заявки на добавление информации новом объекте НСИ можно либо из панели навигации предназначенной для работы с основными данными справочника, либо из панели навигации предназначенной для работы с бизнес процессами и задачами. В результате вызова данной функции пользователю будет предложено заполнить разработанную форму запроса на добавление нового элемент справочника.

Реквизиты:

Задание

Данный реквизит формируется автоматически системой с учетом краткого наименования объекта НСИ.

Справочник

Поле выбора из списка справочников НСИ. Реквизит определяет в какой справочник будет добавлен новый объект НСИ.

Автор

Данное поле заполняется автоматически логином пользователя системы.

В зависимости от указанного пользователем справочника ему будет предложено заполнить различный набор атрибутов (полей) справочника. В момент заполнения атрибутов справочника в системе создается временный элемент справочника НСИ, который станет доступен пользователям только после обработки записи экспертами службы НСИ.

Важно отметить, что система автоматически подсвечивает поля обязательные для заполнения, а также в системе настроены автоматические проверки корректности заполнения тех или иных атрибутов (например, ИНН и КПП в справочнике Контрагентов).

Заполнив предложенную форму заявки на добавлении нового элемента справочника НСИ пользователь сохраняет свою заявку и направляет ее на рассмотрение в службу ведения НСИ.

Старший эксперт службы ведения НСИ принимает заявки на добавление новых объектов НСИ в обработку и распределяет заявки по экспертам в зависимости от предмета заявки.

В случае возникновения у эксперта службы НСИ вопросов, относительно однозначного определения объекта НСИ пользователь получает по средствам системы уточняющий запрос. При получении уточнения информации пользователем достаточного для однозначной идентификации объекта НСИ эксперт службы НСИ продолжает обработку заявки согласно утвержденному бизнес-процессу предприятия. В ходе обработки заявки на добавление нового элемента справочника эксперт службы НСИ нормализует краткое и полное наименование, классифицирует запись в соответствие с иерархическими классификаторами и заполняет все необходимые поля записи. Запись должна быть обязательно проверена экспертом службы НСИ на дублирование и в случае необходимости проставлены связи между выявленными дубликатами. По завершении обработки эксперт заполняет таблицу маппинга и направляет обработанный запрос на утверждение методологу. Методолог просматривает запрос и если данные справочника заполнены корректно и соответствуют корпоративному стандарту и утверждает запрос. Утвержденный запрос автоматически становиться полноценной записью справочника НСИ.
